# Storage Status

## Current State

Bunny storage migration is already active for new uploads.

The current upload provider is:

- `bunnyStorageProvider` from `src/lib/bunny.server.ts`

New dashboard uploads call Bunny directly from:

- `createVideoFromForm`
- `updateVideoFromForm`

## What Still Uses Local Files

The repository still contains local storage support, but it is legacy or
supporting state, not the active upload provider.

Local files still used by the current code:

- `storage/users/users.json` for seeded users
- `storage/metadata/videos.json` for the media catalog

Legacy local media code still present:

- `LocalStorageProvider`
- local upload helper functions
- seed media URLs under `/storage/*`
- local cleanup for old local URLs

## Deployment Target

The deployment target is Vercel.

Build output is generated by TanStack Start, Vite, and Nitro into:

- `.vercel/output`

The app should not be deployed as a static Vite SPA and should not point Vercel
at `dist/server` or `dist/client`.

## Notes

Do not plan another Bunny migration from local media storage. That migration is
already reflected in the active upload code.
